Aktive Entwicklung

Neue Features und Fixes kommen regelmäßig – gebaut von Leuten, die diese Roller selbst fahren.

Getragen von der Community

LibreScoot wird von seinen Nutzer*innen gebaut. Wer den Code schreibt, fährt die Roller täglich. Features entstehen, weil jemand sie gebraucht hat, nicht weil ein Product Manager sie eingeplant hat.

  • Regelmäßige Releases auf dem Testing- und Nightly-Kanal
  • Offene Entwicklung mit öffentlichen Issues, PRs und Discord-Diskussionen

Release-Kanäle

KanalBeschreibung
StableEmpfohlen für die meisten Nutzer*innen. Gründlich getestete Releases.
TestingFrüher Zugriff auf neue Features mit grundlegender Validierung.
NightlyAktuelle Builds aus den Main-Branches. Für Entwickler*innen.

Mitmachen

Die Codebase ist überwiegend Go (Services) und Qt/C++ (Dashboard-UI). Jeder Service liegt in seinem eigenen Repository mit Standard-Build-Tooling.
