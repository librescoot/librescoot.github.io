Navigation Setup

Map display and routing are two independent features, each configurable as offline or online. You can mix and match freely.

Two independent choices

Offline (local) Online (internet)
Map display Local .mbtiles file Online tile server
Routing Local Valhalla + routing maps Remote Valhalla server

Map Display

The map display setting controls which tile source is used to draw the map on screen. It has no bearing on whether you can navigate — that's routing (below).

Switch between modes: Menu → Settings → Map & Navigation → Map Type

Offline display tiles

Offline display uses a local .mbtiles file (an SQLite database of pre-rendered map tiles). Once installed, it works without any internet connection.

Get a tile file for your region:

  • LibreScoot OSM Tiles releases — pre-packaged regional extracts optimised for the scooter display
  • Or generate your own with maptool

The tile file must be placed at /data/maps/map.mbtiles on the scooter. The easiest way to transfer it is UMS mode (see below).

Online display tiles

Online display fetches tiles from an internet tile server as you pan the map. No local file needed — just an active internet connection. Selected automatically if no local tile file is present and the map type is set to Online.

Routing

Routing provides turn-by-turn navigation. It is completely independent of which tile source you use for display. LibreScoot uses the Valhalla routing engine.

Local (offline) routing

Local routing runs Valhalla directly on the scooter's dashboard computer. It requires Valhalla routing map data — these are separate from the display .mbtiles file. Routing data contains the road graph that Valhalla needs to calculate routes.

Routing data is included in the LibreScoot OSM Tiles releases alongside the display tiles, or can be generated separately with maptool.

Routing data is stored at /data/valhalla/ on the scooter. After adding routing data, the Valhalla service needs to be running:

ssh deep-blue "ssh root@192.168.7.2 'systemctl status librescoot-valhalla'"

When the local Valhalla service is up and has data, the Navigation menu becomes available on the dashboard automatically.

Remote (online) routing

You can point LibreScoot at a remote Valhalla server instead of running one locally. This is useful if you don't want to store routing data on the scooter, or are running a shared Valhalla instance.

Configure the endpoint via Redis:

redis-cli -h 192.168.7.2 hset settings dashboard.valhalla-url "https://your-valhalla-server.example/"

The default is http://localhost:8002/ (local Valhalla).

Transferring files via USB (UMS mode)

USB Mass Storage mode mounts the scooter's /data partition as a USB drive on your computer — the easiest way to copy map files.

  1. Power on the scooter and wait for the dashboard
  2. Open the menu (hold right brake) → Settings → System → Enter UMS Mode
  3. Connect a USB-A to USB-A cable between the scooter's USB port and your computer
  4. The scooter storage appears as a drive on your computer
  5. Copy map.mbtiles to maps/ and/or Valhalla routing data to valhalla/
  6. Safely eject — the scooter exits UMS mode and reboots

SSH transfer (advanced)

# Copy display tiles
tar -cf - map.mbtiles | ssh deep-blue "ssh root@192.168.7.2 'mkdir -p /data/maps && tar -xf - -C /data/maps'"

# Copy Valhalla routing data
tar -cf - valhalla/ | ssh deep-blue "ssh root@192.168.7.2 'mkdir -p /data/valhalla && tar -xf - -C /data'"

# Restart services after copying
ssh deep-blue "ssh root@192.168.7.2 'systemctl restart librescoot-mbtileserver librescoot-valhalla'"
