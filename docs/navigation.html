Navigation Setup

LibreScoot supports two navigation modes: offline routing with local map tiles and a local routing engine, and online map display using tile servers over the internet.

Quick summary

  • Offline navigation: Full turn-by-turn routing, works without internet. Requires a map tile file and the Valhalla routing engine.
  • Online maps: Map display only, no routing. Works automatically when internet is available. No setup needed.

Offline Navigation

Offline navigation uses OpenStreetMap data stored locally on the scooter. Once set up, it works entirely without an internet connection — routing included.

What you need

  1. A map tile file (.mbtiles) for your region
  2. The Valhalla routing engine (included with LibreScoot — just needs map data)
  3. A computer with a USB cable to transfer the tile file

Step 1: Download map tiles

Map tiles are distributed as .mbtiles files (SQLite databases containing pre-rendered map tiles). Download a file for your region:

  • LibreScoot OSM Tiles releases — pre-packaged regional extracts optimised for the scooter display
  • Or generate your own using maptool

The tile file must end in .mbtiles. Region extracts are typically 200 MB–2 GB depending on area size.

Step 2: Transfer map tiles via USB

The easiest way to transfer files to the scooter is USB Mass Storage (UMS) mode. This mounts the scooter's storage as a USB drive on your computer.

  1. Make sure the scooter is powered on and the dashboard is showing
  2. Open the menu (hold right brake) → Settings → System → Enter UMS Mode
  3. Connect a USB-A to USB-A cable between the scooter's USB port and your computer
  4. The scooter storage will appear as a drive on your computer
  5. Copy your .mbtiles file to the maps/ folder on the drive
  6. Safely eject the drive, then the scooter will reboot automatically

The maps/ directory is located at /data/maps/ on the scooter. If it doesn't exist yet, create it.

Step 3: Verify setup

After rebooting, open the menu. The Navigation entry should now appear. If it doesn't:

  • Check that the file is named *.mbtiles and is in the maps/ folder
  • Check that the Valhalla service is running: ssh deep-blue "systemctl status librescoot-valhalla"
  • Valhalla needs to index the routing data on first boot after adding tiles — this can take a few minutes

Online Maps

When an internet connection is available, LibreScoot can display online maps without any local tile files. Online maps use OpenStreetMap tile servers.

To switch to online maps: Menu → Settings → Map & Navigation → Map Type → Online

Online maps do not include routing. Turn-by-turn navigation always requires the local Valhalla routing engine, even when using online tiles. If you only want to view a map while riding, online mode works fine without any setup.

Advanced: SSH Transfer

If you have SSH access to the scooter, you can transfer map files directly without UMS mode:

# Copy a tile file to the scooter
tar -cf - myregion.mbtiles | ssh deep-blue "ssh root@192.168.7.2 'mkdir -p /data/maps && tar -xf - -C /data/maps'"

# Verify it's there
ssh deep-blue "ssh root@192.168.7.2 'ls -lh /data/maps/'"

After copying, restart the tile server:

ssh deep-blue "ssh root@192.168.7.2 'systemctl restart librescoot-mbtileserver'"

Routing Data for Valhalla

The Valhalla routing engine needs separate routing graph data — distinct from the display tile files. LibreScoot packages both together in the recommended tile releases.

If you're generating custom tiles with maptool, it will produce both the .mbtiles display file and the Valhalla routing tiles in one step.

Routing tiles are stored at /data/valhalla/ on the scooter.
