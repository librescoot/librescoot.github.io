{%- comment -%} Custom redirect layout that's polyglot-aware. jekyll-redirect-from's default template emits the target URL in three places: the canonical , a , and a meta refresh. Polyglot's relativize_urls pass only rewrites href= attributes, so without this layout the canonical lands at /en/... but the script and meta refresh still point at the default-lang URL — sending EN users to DE content. Here we compute target_url once, lang-aware, and use it everywhere. {%- endcomment -%} {%- assign target_url = page.redirect.to -%} {%- if site.active_lang != site.default_lang -%} {%- assign lang_segment = "/" | append: site.active_lang | append: "/" -%} {%- unless target_url contains lang_segment -%} {%- assign target_url = target_url | replace: site.url, "" -%} {%- capture target_url -%}{{ site.url }}{{ lang_segment | replace_first: "//", "/" }}{{ target_url | remove_first: "/" }}{%- endcapture -%} {%- endunless -%} {%- endif -%} Redirecting…
